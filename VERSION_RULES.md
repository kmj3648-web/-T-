# 프로젝트 버전 관리 및 AI 작업 규칙 (AI Update Guidelines)

이 파일은 AI가 코드를 수정하고 시스템을 업데이트할 때 **반드시 준수해야 하는** 버전 관리 및 패치노트 작성 규칙입니다. 세션이 재시작되더라도 이 파일을 우선적으로 읽고 아래의 지침을 엄격하게 따라야 합니다.

## 1. 버전 넘버링 규칙 (Semantic Versioning)
코드를 수정할 때마다 변경 사항의 규모에 따라 버전을 산정합니다.
- **Major 업데이트 (X.0.0)**: 데이터베이스(DB) 스키마 갱신, 완전히 새로운 기능/페이지 추가 등 대대적인 패치
- **Minor 업데이트 (O.X.0)**: 기존 페이지 내 기능 추가, 기능성 개선 등 마이너 패치
- **Patch 업데이트 (O.O.X)**: 단순 버그 수정, 오타 교정, 사소한 로직 변경 등 가벼운 패치

## 2. 문서 업데이트 및 패치노트 작성
업데이트가 발생하여 버전이 올라갈 때마다 다음 문서화 작업을 수행해야 합니다.
1. **유저 가이드 파일명 갱신**: 기존 `USER_GUIDE_vX.Y.Z.md` 파일의 이름을 새 버전에 맞게 변경합니다.
2. **패치노트 기재**: 유저 가이드(`USER_GUIDE_vX.Y.Z.md`) 문서의 맨 하단에 **[패치노트 (Release Notes)]** 섹션을 두고, 업데이트된 버전명과 함께 변경된 내용을 상세히 기록합니다.

## 3. 프로젝트 폴더명 갱신
버전이 업데이트될 경우, 현재 작업 중인 최상위 폴더의 이름도 버전에 맞게 변경해야 합니다.
- **형식 규칙**: `백업용 파일 (X.Y.Z)`
- 코드를 수정 완료한 후, 터미널 명령어나 스크립트를 사용하여 최상위 폴더 이름을 새 버전에 맞게 즉각적으로 Rename 합니다.

**[AI 에이전트를 위한 Action Item]**
사용자(원장님)의 요청으로 코드를 수정했다면, 작업 완료 보고를 하기 전에 반드시 다음 3단계를 점검하라.
1. 어떤 스케일의 패치인지 판단하여 새 버전 넘버를 확정했는가?
2. `USER_GUIDE` 파일명을 바꾸고 하단에 패치노트를 작성했는가?
3. 최상위 폴더 `백업용 파일 (X.Y.Z)`의 이름을 새 버전에 맞게 변경했는가?
